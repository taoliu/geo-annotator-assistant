# v0.4 — Curator-Ready Backend

## Summary

v0.4 completes the transition from a fully automated GSM annotation pipeline to a **curator-ready backend**, without introducing UI or changing the core inference architecture.

This release focuses on:

* performance stability at scale,
* explicit human-in-the-loop correction,
* structured review artifacts,
* and opt-in, non-forcing cross-GSM diagnostics.

All changes are incremental, deterministic, and auditable.

---

## Goals

The primary goals of v0.4 were:

1. Make GSM-level results **inspectable and reviewable** by humans.
2. Enable **explicit human corrections** without breaking determinism.
3. Preserve all v0.x invariants:

   * fixed output schema,
   * no hidden state,
   * no forced label propagation,
   * no implicit learning.
4. Improve runtime performance for large GSM batches on a single GPU.

---

## Major Additions

### 1. Performance stabilization (single GPU)

* Local HuggingFace LLM is now loaded **once per process** and reused across all GSMs.
* Eliminates repeated model initialization overhead.
* No change to inference logic or outputs.

Result:

* GSE-scale runs are practical on a single GPU.
* Backend is responsive enough for iterative curation workflows.

---

### 2. Explicit human override mechanism

* Introduced `overrides.jsonl` as the **only supported mechanism** for human corrections.
* Overrides are:

  * explicit inputs,
  * deterministic,
  * auditable,
  * non-interactive.

Key properties:

* Overrides apply **after** inference and repair.
* No re-inference or ontology re-grounding is triggered.
* All applied overrides are recorded in audit metadata with old and new values.

---

### 3. Structured curator review artifacts

v0.4 formalizes a set of machine-readable review outputs:

#### `curation.tsv`

* Continues to provide curator-friendly tabular output.
* Semantics unchanged from v0.3.

#### `curation.jsonl`

* Lossless JSON mirror of `curation.tsv`.
* Derived from the same row-building logic.
* Uses native JSON types instead of TSV stringification.

#### `evidence.jsonl`

* Structural diagnostic evidence only.
* Surfaces existing audit signals:

  * repair attempts,
  * terminal fallback usage,
  * ontology grounding status,
  * field-relevant flags.
* No free-text rationale or new inference.

These artifacts together form a **stable review bundle** suitable for UI and downstream tooling.

---

### 4. Opt-in cross-GSM suggestions (advisory only)

* Added a deterministic aggregation pass to emit `suggestions.jsonl`.
* Suggestions:

  * identify outliers, singletons, and majority disagreements within a GSE,
  * are emitted only when explicitly requested,
  * never modify GSM outputs.

This preserves GSM independence while helping curators prioritize review.

---

## Non-Goals (Explicitly Deferred)

v0.4 intentionally does **not** include:

* Curator UI
* Persistent databases or hidden state
* Automatic learning from human edits
* Forced cross-GSM label propagation
* Ontology validation of override values

These are deferred to later releases.

---

## Stability and Compatibility

* No breaking changes to v0.3 output schema.
* Existing TSV-based workflows continue to work.
* All new functionality is opt-in or additive.
* Pipeline behavior is unchanged when new features are not used.

---

## Artifacts Produced

A v0.4 run may produce:

* `curation.tsv`
* `curation.jsonl`
* `evidence.jsonl`
* `suggestions.jsonl` (opt-in)
* `audit.jsonl`

And may consume:

* `overrides.jsonl` (optional, explicit input)

---

## Milestone Assessment

v0.4 completes the **backend contract** required for human-in-the-loop curation.

At this point:

* The system is fully scriptable and reproducible.
* All human interaction points are explicit and auditable.
* The backend is ready to support one or more curator UIs without further architectural changes.

This milestone intentionally stops at the backend boundary.

## Next Milestone

**v0.5 — Curator UI**
