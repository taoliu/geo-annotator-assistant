# Milestone v0.2 — Ontology Grounding and Evidence-First Repair

## Scope

This milestone focused on making ontology grounding and validation mechanically correct, deterministic, and compatible with a repair-first labeling loop.

The goal was correctness of *process*, not final labeling policy.

---

## Completed Work

### Ontology integration
- Integrated ChromaDB-backed ontology retrieval.
- Supported Uberon, DOID, EFO, Cellosaurus, NCIT via a unified collection.
- Implemented exact, ambiguous, low-confidence, and fallback outcomes.

### Decision engine stabilization
- Introduced deterministic primary failure selection.
- Added evidence-first short-circuit for hallucinated fields.
- Ensured ontology confidence failures do not mask semantic evidence errors.

### Repair loop correctness
- Implemented bounded, field-specific repair prompts.
- Verified multi-step repair convergence.
- Ensured audit logs reflect actual reasoning path.

### Testing and tooling
- Added missing runtime dependencies.
- Standardized testing under `uv run pytest -q`.
- Added regression tests for failure priority selection.

---

## Explicit Non-Goals

- Defining canonical semantics for missing values (`Unknown` vs `Healthy`).
- Changing prompt language beyond repair mechanics.
- Optimizing ontology thresholds or confidence heuristics.

These are deferred to the next milestone.

---

## Exit Criteria (Satisfied)

- Ontology grounding functions correctly end-to-end.
- Hallucinated values are removed before ontology enforcement.
- Pipeline converges deterministically on valid inputs.

---

## Next Milestone

v0.3 will address **labeling policy**, including canonical handling of missing information and evidence-aware defaults.
