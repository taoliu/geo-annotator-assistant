# Milestone v0.5 — Curator UI and Review Workflow

## Goal

Provide a local, deterministic curator interface for inspecting and correcting
GEO GSM annotations produced by the v0.4 backend, without introducing any
hidden state, persistence, or learning.

## Scope (Delivered)

### UI Capabilities

* Local Streamlit-based curator UI
* Read-only by default
* Loads:
  * curation.jsonl
  * evidence.jsonl
  * suggestions.jsonl (optional)
* Wide, searchable GSM table
* Per-GSM detail panels:
  * raw curation record
  * raw evidence diagnostics
  * raw suggestions (advisory only)

### Field-Level Review Support

* Field-level issue highlighting derived from evidence.jsonl
* Deterministic flag extraction
* No inference or interpretation added

### Editing Workflow

* Explicit edit mode (opt-in)
* Inline table editing for canonical fields:
  * data_type
  * organism
  * tissue_type
  * cell_line
  * disease
  * treatment
* All edits are session-scoped (in memory only)
* Clear per-GSM and clear-all edit controls

### Overrides Export

* Explicit export of overrides.jsonl
* Backend-compatible schema:
  * gsm_accession
  * field
  * new_value
* Deterministic ordering
* Preview-before-export
* No automatic writes

## Non-Goals (Explicitly Preserved)

* No backend pipeline changes
* No inference, repair, or grounding
* No persistence or learning
* No automatic suggestion application
* No hidden state

## Outcome

v0.5 completes the curator-facing review loop while preserving all v0.x
architectural invariants.

Milestone closed.
