# Milestone v1.1 — Curator UI & UX Refinement

## Scope and Intent

Milestone v1.1 continues work after **v1.0 Curator UI**, focusing exclusively on
**curator-facing UI and UX refinement** based on real usage.

This milestone is intentionally **UI-only**.

- Backend behavior is frozen and authoritative as of **v0.9**
- No validation, repair, ontology, or decision logic was modified
- No schema changes were introduced

The goal of v1.1 is to make the system efficient, transparent, and safe for
large-scale, long-session human curation.

---

## Design Constraints (Reaffirmed)

- Backend output remains the source of truth
- UI reflects state; it does not infer or fix
- Evidence is shown, not hidden
- All curator actions are explicit and reversible
- UX clarity is prioritized over feature count

---

## Major Improvements

### 1. Curation Table Interaction Model

- Migrated the curation table to **AG Grid** for robust interaction support
- Introduced pinned workflow columns:
  - **Status** (icon-only)
  - **Checked** (persistent curator marker)
  - **Edited** (live pencil indicator)
- Workflow columns are:
  - compact
  - icon-based
  - pinned to the left
  - non-movable
- Editing is **always enabled**; no edit-mode toggle exists
- Modal-based detail inspection was removed

---

### 2. Evidence-Centric Transparency

- All per-field diagnostics are sourced exclusively from **`evidence.jsonl`**
- Cell highlighting is driven **only** by explicit evidence flags:
  - fallback states are informational, not warnings
- Redundant diagnostic columns were removed:
  - review flags
  - terminal fallbacks
  - outliers
  - primary failure
  - flag summary
  - flagged_fields
- Field-level inspection is provided via rich hover tooltips, including:
  - displayed value
  - backend-derived value
  - LLM raw proposal
  - ontology alternates (when present)
  - evidence attempts, ontology status, terminal fallback, and flags

This eliminates duplicated signals and aligns UI semantics with policy definitions.

---

### 3. Curator Workflow Enhancements

- Added a persistent **Checked** column:
  - curator-controlled
  - saved to disk as a UI-only artifact
  - does not affect backend decisions
- Fixed and stabilized the **Edited** indicator:
  - updates live on cell edit
  - clears correctly on revert operations
  - reflects both session edits and saved overrides
- Introduced **bulk editing**:
  - explicit row selection
  - single-column, single-value apply
  - preview before apply
  - full reversibility using existing edit controls
- Implemented progressive disclosure for persistence actions:
  - **Save overrides** always visible
  - **Revert to saved** and **Discard saved overrides** hidden under a secondary control

---

### 4. Export and Reporting Improvements

- Added CSV export for:
  - GSE-wide biology summary
  - per-GSE final annotations
- Added **aggregate CSV export** across all GSMs loaded under an `input-dir`
- All exports:
  - include exactly the 8 canonical output fields
  - apply saved overrides
  - do not trigger backend reprocessing

---

## Explicit Non-Changes

- No backend code paths were modified
- No validation, repair, or ontology rules were changed
- No new flags or failure codes were introduced
- No inference or normalization logic was added to the UI
- No cross-GSM propagation or automation was introduced

---

## Known Limits (Intentionally Deferred)

- Single-user UI (no collaboration or locking)
- No long-lived multi-user sessions
- No CLI ergonomics changes (planned for next milestone)
- No backend performance changes

---

## Outcome

At the conclusion of v1.1:

- The curator UI is production-ready for large GSEs
- High-volume curation is efficient and safe
- Diagnostics are field-scoped, evidence-based, and non-redundant
- Reporting outputs are directly usable without post-processing

This milestone closes the UI refinement phase and hands off cleanly to
**command-line and batch workflow refinement** in the next milestone.
